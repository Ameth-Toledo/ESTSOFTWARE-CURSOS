Comando SUDO CHMOD

El comando sudo chmod se utiliza para cambiar los permisos de un archivo o directorio en Linux. Los permisos definen quién puede leer, escribir o ejecutar un archivo. Los permisos se pueden establecer utilizando tanto letras como números.

En lugar de utilizar letras como r (lectura), w (escritura) y x (ejecución), puedes usar números. Los permisos se asignan usando un sistema de tres dígitos donde cada dígito representa una combinación de permisos para el propietario, el grupo y otros usuarios.

Explicación de los números en CHMOD

Los números utilizados en chmod están basados en el sistema octal, donde:

  • 4 = Permiso de lectura (r)
  • 2 = Permiso de escritura (w)
  • 1 = Permiso de ejecución (x)
  • 0 = Sin permisos

Los números se suman para determinar los permisos. Por ejemplo:

  • 7 = 4 (lectura) + 2 (escritura) + 1 (ejecución) = rwx (lectura, escritura y ejecución)
  • 6 = 4 (lectura) + 2 (escritura) = rw- (lectura y escritura)
  • 5 = 4 (lectura) + 1 (ejecución) = r-x (lectura y ejecución)
  • 4 = 4 (lectura) = r-- (solo lectura)
  • 3 = 2 (escritura) + 1 (ejecución) = wx- (escritura y ejecución)
  • 3 = 2 (escritura) + 1 (ejecución) = wx- (escritura y ejecución)
  • 2 = 2 (escritura) = w-- (solo escritura)
  • 1 = 1 (ejecución) = x-- (solo ejecución)
  • 0 = Sin permisos = ---

Así, cuando se utiliza sudo chmod 777 archivo.txt, se le otorgan al propietario, al grupo y a los demás usuarios los permisos completos (lectura, escritura y ejecución) sobre el archivo.

Ejemplos prácticos:

  • sudo chmod 755 archivo.txt: Asigna permisos de lectura, escritura y ejecución al propietario, y solo lectura y ejecución al grupo y otros.
  • sudo chmod 644 archivo.txt: Asigna permisos de lectura y escritura al propietario, y solo lectura al grupo y otros.
